🏗️ Architecture Full-Stack

Le projet se compose de deux applications distinctes : un serveur backend Node.js/Express exposant une API REST sécurisée par JWT, et une application mobile Flutter utilisant GetX pour la gestion d'état. Les deux communiquent via HTTP avec un système de renouvellement automatique des tokens.

03 — Base de Données

Modèle de Données

11 tables MySQL avec relations complexes via clés étrangères et tables de jointure, gérées par Sequelize ORM.

Modèle Champs Principaux Relations Rôle
User name, email, password, googleId, facebookId, appleId, age, weight, height, gender, fitnessLevel, split, daysPerWeek, location hasMany WorkoutHistory Profil utilisateur & préférences
Exercise name, force, level, mechanic, equipment, category belongsToMany Muscle, WorkoutTemplate; hasMany Gallery, Instruction Catalogue d'exercices
Muscle name (unique) belongsToMany Exercise Groupes musculaires
ExerciseMuscle role ("primary" | "secondary") Jointure Exercise ↔ Muscle Ciblage musculaire
Gallery link, type (image | video) belongsTo Exercise Médias d'exercices
Instruction text belongsTo Exercise Instructions textuelles
WorkoutTemplate name, dayOfWeek, focus, category, split, fitnessLevel, location, isRestDay, estimatedDurationMinutes belongsToMany Exercise; hasMany WorkoutHistory Plans d'entraînement prédéfinis
WorkoutTemplateExercise orderIndex, sets, reps, restSeconds, notes Jointure WorkoutTemplate ↔ Exercise Configuration exercice/template
WorkoutHistory userId, workoutTemplateId, dateAssigned, completed belongsTo User, WorkoutTemplate; hasMany WorkoutHistoryExercise Sessions d'entraînement
WorkoutHistoryExercise sets, reps, restSeconds, notes, performedSets (JSON), completed belongsTo WorkoutHistory Progression par exercice
VerificationCode code, email/phone, expiresAt — Vérification email/téléphone

🔗 Relations Clés

User → WorkoutHistory : Un utilisateur a plusieurs sessions. WorkoutTemplate → Exercise : Many-to-Many via table de jointure avec métadonnées (sets, reps, ordre). Exercise → Muscle : Many-to-Many avec rôle (primaire/secondaire). WorkoutHistory → WorkoutHistoryExercise : Suivi granulaire de la performance par exercice avec données JSON des sets réalisés.

📊 Structure des Données de Performance

Chaque exercice réalisé stocke un tableau JSON de sets effectués :

Documentation API REST

Méthode Endpoint Authentification Description

🏋️ Routes Workout — /workouts

Méthode Endpoint Paramètres Description
GET /workouts/recommended — Templates recommandés selon le profil utilisateur
GET /workouts/today — Récupère ou crée la session du jour avec exercices
GET /workouts/stats from, to, days, topMusclesLimit Statistiques complètes avec gamification et tendances
GET /workouts/history limit, offset Historique paginé des sessions passées
GET /workouts/history/:id workoutHistoryId Détail d'une session avec progression par exercice
PUT /workouts/history/:id/exercises/:exerciseId performedSets, completed Mise à jour de la progression d'un exercice
POST /workouts/history/:id/finish workoutHistoryId Marquer un workout comme terminé

📊 Summary

Sessions terminées, exercices complétés, total sets/reps, volume total, durée moyenne

🎮 Gamification

Points accumulés, niveau actuel, seuil prochain niveau, pourcentage de progression

📈 Tendances

Comparaison 7 jours glissants vs 7 jours précédents avec indicateurs de progression

💪 Top Muscles

Classement des muscles les plus travaillés avec score et jours d'entraînement

06 — Intelligence

Moteur de Workout Intelligent

Algorithme sophistiqué d'attribution et de recommandation des séances d'entraînement basé sur le profil utilisateur.

Splits Supportés

Push / Pull / Legs

Séparation par mouvement — poussée, tirage, jambes. Idéal pour 3-6 jours/semaine.

Upper / Lower

Alternance haut du corps et bas du corps. Adapté pour 4 jours/semaine.

Full Body

Entraînement complet à chaque session. Optimal pour 2-3 jours/semaine.

Bro Split

Un groupe musculaire par jour. Pour les pratiquants avancés, 5-6 jours/semaine.

Logique de Sélection (5 niveaux)

TIER 1 Match strict

Split + catégorie + niveau + lieu + durée, sans répétition de template récent

TIER 2 Ignorer la durée

Mêmes critères sans contrainte de durée estimée

TIER 3 Autoriser les répétitions

Permet de réutiliser des templates déjà assignés récemment

TIER 4 Catégorie élargie

N'importe quel template du split et de la catégorie

TIER 5 Fallback

N'importe quel template non-repos disponible

🎮 Système de Gamification

Les utilisateurs accumulent des points basés sur les sessions complétées, le volume total et la régularité. Le système calcule un niveau avec un seuil de progression vers le niveau suivant, encourageant l'engagement continu. Les tendances sur 7 jours glissants comparent la performance actuelle avec la semaine précédente.
