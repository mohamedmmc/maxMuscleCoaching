🏗️ Full-Stack Architecture

The project consists of two distinct applications: a Node.js/Express backend server exposing a JWT-secured REST API, and a Flutter mobile app using GetX for state management. Both communicate over HTTP with an automatic token renewal system.

03 — Database

Data Model

11 MySQL tables with complex relationships via foreign keys and join tables, managed by Sequelize ORM.

Model Key Fields Relationships Purpose
User name, email, password, googleId, facebookId, appleId, age, weight, height, gender, fitnessLevel, split, daysPerWeek, location hasMany WorkoutHistory User profile & preferences
Exercise name, force, level, mechanic, equipment, category belongsToMany Muscle, WorkoutTemplate; hasMany Gallery, Instruction Exercise catalog
Muscle name (unique) belongsToMany Exercise Muscle groups
ExerciseMuscle role ("primary" | "secondary") Join: Exercise ↔ Muscle Muscle targeting
Gallery link, type (image | video) belongsTo Exercise Exercise media
Instruction text belongsTo Exercise Exercise instructions
WorkoutTemplate name, dayOfWeek, focus, category, split, fitnessLevel, location, isRestDay, estimatedDurationMinutes belongsToMany Exercise; hasMany WorkoutHistory Pre-designed workout plans
WorkoutTemplateExercise orderIndex, sets, reps, restSeconds, notes Join: WorkoutTemplate ↔ Exercise Exercise/template config
WorkoutHistory userId, workoutTemplateId, dateAssigned, completed belongsTo User, WorkoutTemplate; hasMany WorkoutHistoryExercise Workout sessions
WorkoutHistoryExercise sets, reps, restSeconds, notes, performedSets (JSON), completed belongsTo WorkoutHistory Per-exercise progress
VerificationCode code, email/phone, expiresAt — Email/phone verification

🔗 Key Relationships

User → WorkoutHistory: A user has multiple sessions. WorkoutTemplate → Exercise: Many-to-Many via join table with metadata (sets, reps, order). Exercise → Muscle: Many-to-Many with role (primary/secondary). WorkoutHistory → WorkoutHistoryExercise: Granular per-exercise performance tracking with JSON set data.

06 — Intelligence

Smart Workout Engine

Sophisticated workout assignment and recommendation algorithm based on user profile and training history.

Supported Splits

Push / Pull / Legs

Movement-based separation — push, pull, legs. Ideal for 3-6 days/week.

Upper / Lower

Alternating upper and lower body. Best suited for 4 days/week.

Full Body

Complete body training each session. Optimal for 2-3 days/week.

Bro Split

One muscle group per day. For advanced lifters, 5-6 days/week.

Selection Logic (5-Tier Fallback)

TIER 1 Strict Match

Split + category + level + location + duration, avoiding recently used templates

TIER 2 Ignore Duration

Same criteria without estimated duration constraint

TIER 3 Allow Repeats

Permits reusing recently assigned templates

TIER 4 Broader Category

Any template matching the split and category

TIER 5 Fallback

Any available non-rest template
